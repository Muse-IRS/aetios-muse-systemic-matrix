# Security Policy

Aétios Muse Systemic Matrix (AMSM)  
Silence-First Framework (SSF)

Status: Public reference repository  
Last update: 2026

---

## Supported Versions

This repository is a reference implementation.

Security fixes may be applied to the latest version only.

| Version | Supported |
|--------:|:---------:|
| latest  |    ✅     |
| older   |    ❌     |

---

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it
responsibly.

This includes, but is not limited to:
- bypass of SSF gate logic,
- denial-of-service vectors (CPU, memory, infinite loops),
- persistence integrity issues (state poisoning, replay),
- injection risks in reference code or examples,
- cryptographic misuse (if applicable).

### Preferred channel

For now, the only supported channel is:

- a **GitHub Security Advisory** (preferred),
- or a **private GitHub message** to the repository owner.

Do **not** open a public issue for active vulnerabilities.

---

## Disclosure Process

1. Report privately.
2. Acknowledge receipt.
3. Validate and reproduce.
4. Patch and publish.
5. Credit is optional and must be explicitly requested.

---

## What This Repository Does Not Provide

This repository does not provide:
- SLA,
- guaranteed response times,
- commercial support commitments.

It is published as an open research initiative.

---

## Safe Harbor

Good-faith security research is welcomed.

However:
- no social engineering,
- no data exfiltration attempts,
- no disruption of third-party services.
